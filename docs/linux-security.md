# Segurança em Linux

## Boas práticas

- Manter pacotes atualizados  
- Utilizar senha forte  
- Desativar login root remoto (SSH)  
- Utilizar firewall (ufw ou iptables)  
- Monitorar logs do sistema  
- Remover serviços desnecessários  
- Utilizar usuários com permissões limitadas  

## Controles importantes

- Gerenciamento de permissões (chmod, chown)  
- Controle de usuários e grupos  
- Configuração do SSH  
- Monitoramento de logs (/var/log)  

## Exemplos de riscos

- Permissões mal configuradas  
- Serviços expostos desnecessariamente  
- SSH aberto sem proteção  
- Sistema desatualizado  

## Importância

A configuração adequada de segurança em sistemas Linux reduz vulnerabilidades e garante maior controle sobre o ambiente.
